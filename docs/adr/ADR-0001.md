# ADR-0001 · Precargar datos vía ETL en vez de llamar APIs en vivo

| Campo          | Valor                                                              |
|----------------|--------------------------------------------------------------------|
| Estado         | Aceptada · 26-06-2026                                             |
| Contexto       | La evaluación exige una URL de producción funcional al momento del pitch. Los portales del Estado pueden estar lentos o caídos, y algunos bloquean acceso automatizado. |
| Decisión       | Hacer una ETL única que descarga OCDS + el anexo de obras y los normaliza a Postgres. La app consulta solo nuestra base. |
| Consecuencias  | **+** Demo estable e independiente de terceros.<br>**−** Datos a una fecha de corte (aceptable para un MVP). |
